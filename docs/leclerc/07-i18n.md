# 07 · Internationalization (locale middleware)

> Pattern lifted from `defi-web-app` (the more mature of the two references). Library: **`next-international@^1.3.1`**. v1 locales: **`en`, `es`** (es is the default given Whisper/MedPsy `language:"es"` config and existing copy). The spy re-theme IS the i18n content — translating the strings re-skins the app.

## 1. Files to create (App Router)

```
apps/app/
├─ proxy.ts                      # next-international middleware (Next.js middleware convention)
├─ messages/
│  ├─ en.json                    # English (spy/intel theme)
│  └─ es.json                    # Spanish (default)
├─ locales/
│  ├─ client.ts                  # createI18nClient
│  └─ server.ts                  # createI18nServer
└─ src/app/[locale]/             # locale segment wraps the app
```

## 2. Supported locales (single source)

`apps/app/src/lib/i18n/supported-locales.ts`:
```ts
export const SUPPORTED_LOCALES = ["es", "en"] as const;
export type SupportedLocale = (typeof SUPPORTED_LOCALES)[number];
export const DEFAULT_LOCALE: SupportedLocale = "es";
export function getSupportedLocale(raw?: string | null): SupportedLocale {
  return (SUPPORTED_LOCALES as readonly string[]).includes(raw ?? "") ? (raw as SupportedLocale) : DEFAULT_LOCALE;
}
```

## 3. Middleware (`apps/app/proxy.ts`)

```ts
import { createI18nMiddleware } from "next-international/middleware";
import { SUPPORTED_LOCALES, DEFAULT_LOCALE } from "@/lib/i18n/supported-locales";

const I18nMiddleware = createI18nMiddleware({
  locales: [...SUPPORTED_LOCALES],
  defaultLocale: DEFAULT_LOCALE,
  urlMappingStrategy: "rewrite",     // hide /es prefix from the URL like defi-web-app
});

export function proxy(request) { return I18nMiddleware(request); }

export const config = {
  matcher: ["/((?!_next|api|.*\\..*).*)"],   // skip _next, api, static files
};
```
> Note: exclude `api` from the matcher — the QVAC/wallet Route Handlers are not localized.

## 4. Client + server config

`apps/app/locales/client.ts`:
```ts
"use client";
import { createI18nClient } from "next-international/client";
export const { useI18n, useScopedI18n, I18nProviderClient, useCurrentLocale, useChangeLocale } =
  createI18nClient({
    es: () => import("../messages/es.json"),
    en: () => import("../messages/en.json"),
  });
```

`apps/app/locales/server.ts`:
```ts
import { createI18nServer } from "next-international/server";
export const { getI18n, getScopedI18n, getCurrentLocale, getStaticParams } =
  createI18nServer({
    es: () => import("../messages/es.json"),
    en: () => import("../messages/en.json"),
  });
```

## 5. Layout wiring (`apps/app/src/app/[locale]/layout.tsx`)

```tsx
import { I18nProviderClient } from "@/locales/client";

export default async function LocaleLayout({ children, params }:{
  children: React.ReactNode; params: Promise<{ locale: string }>;
}) {
  const { locale } = await params;
  return (
    <html lang={locale}>
      <body>
        <I18nProviderClient locale={locale}>{children}</I18nProviderClient>
      </body>
    </html>
  );
}
```
Keep a thin root `apps/app/src/app/layout.tsx` (`<html lang="es">` shell) and move the real shell into `[locale]/layout.tsx`, as defi-web-app does.

## 6. Consumption

```tsx
// client component
import { useI18n } from "@/locales/client";
const t = useI18n();  // t("brief.title")

// server component
import { getI18n } from "@/locales/server";
const t = await getI18n();
```

## 7. Locale switcher (`apps/app/src/components/locale-switcher.tsx`)
Port from `defi-web-app/apps/web/components/locale-switcher/index.tsx` — `useChangeLocale` + `useCurrentLocale`, with flags/native names (`🇬🇧 English`, `🇪🇸 Español`).

## 8. Message namespaces (suggested keys)

Mirror screens: `nav.*`, `capture.*`, `dossier.*`, `record.*`, `brief.*`, `wallet.*`, `p2p.*`, `settings.*`, `threat.{CRITICO,ELEVADO,RUTINARIO}`, `kind.*`. Threat-level and record-kind **enums stay constant in data**; only their *labels* are translated. EN strings carry the spy theme ("Dossier", "Asset", "Dead-drop", "Station"); ES the field-intel equivalents ("Expediente", "Contacto", "Buzón", "Estación").

## 9. Acceptance criteria

- [ ] `/` serves `es` by default; switcher flips to `en` and persists (cookie), URL stays clean (rewrite strategy).
- [ ] All visible copy comes from `messages/*.json` — no hardcoded UI strings in components.
- [ ] `api/*` routes are excluded from the i18n matcher and still work.
- [ ] Whisper/QVAC `language` follows the active locale.
